Curso de HTML

HTML es el lenguaje encargado de dar contenido y estructura a la web. Texto, imágenes, videos, etc.

CSS es el lenguaje que da colores, tamaños y diseño en general a las páginas web.

Los lenguajes de programación como Python y Javascript nos permiten agregar acciones a nuestras páginas web o programas, más interacción por parte del usuario. Un ejemplo de esto pueden ser los botones que nos permiten cambiar una página de estilo de día para estilo de noche o viceversa.

Los navegadores web ya entienden estas tecnologías, no necesitamos instalar nada. Se les conoce como estándares porque todos los navegadores web utilizan estas tecnologías.

Estos lenguajes estándares están regidos por grupos de personas que forman parte de grandes tecnologías, y son los que van delimitando cuáles son los avances que los navegadores tienen que ir aceptando en sus nuevas versiones para darnos más características a los programadores y desarrolladores web.

Las personas encargadas de la definición de los estándares de HTML son la W3C. Podemos entrar a su página desde el link: https://www.w3.org/. La documentación de la W3C es muy amplia y tediosa de leer.

Además de la W3C, con el tiempo salió otro grupo de personas que se dedica a definir los estándares, son la Whatwg, podemos ver su documentación en el siguiente link: https://whatwg.org/. Sin embargo desde hace algunos años colaboran con la W3C. Es recomendable leer la W3C en lugar de la whatwg, es prácticamente la documentación oficial, es más amigable.

Otro sitio recomendable para aprender sobre HTML y su documentación es MDN en el siguiente link: . Aunque no es la documentación oficial es muy utilizada por los programadores y es una documentación proporcionada por los creadores de Mozilla. Tienen documentación para HTML, CSS y Javascript.

Otra página recomendable para consultar las funciones de las etiquetas de HTML es HTML reference, podemos verla en el siguiente link: https://htmlreference.io/. Nos da algunos enlaces a documentaciones, ejemplos e incluso nos menciona los atributos que puede tener una etiqueta.

¿Qué es HTML?

HTML (hypertext markup language) es un lenguaje de marcas basado en etiquetas.

Estructura básica de HTML

Este documento ya posee una estructura básica de HTML, procederemos a comprender para qué funciona cada una de sus línea.

Las etiquetas HTML comienzan con el símbolo de menor qué (<) y terminan con el símbolo de mayor que (>), conocidos como picoparéntesis. Hay etiquetas que abren y cierran, tienen apertura () y cierre (), el cierre tiene un slash (/) luego del menor que. También hay etiquetas que tienen una etiqueta de apertura solamente y son mejor conocidas como selfclosing.

Las etiquetas HTML pueden tener atributos, más adelante se tratarán a profundidad en este curso, los atributos afectan las etiquetas de formas específicas y tienen su propia función.

Las etiquetas y atributos se escriben en minúsculas como buena práctica (excepto por la etiqueta DOCTYPE), aunque HTML no distingue mayúsculas y minúsculas.

La etiqueta principal de un documento es html, la cual está compuesta por dos etiquetas más, la etiqueta head y la etiqueta body. Todo lo que esté dentro de la etiqueta head es para enlazar otro tipo de archivo como las hojas de estilos, archivos Javascript, algunas imágenes que sirven a los sitios web para los íconos que se instalan en los navegadores o dispositivos móviles (el favicon que es el ícono que se muesta en las pestaña donde abrimos la página), títulos y descripciones que tienen que ver con el SEO de las páginas en los motores de búsqueda, etc.

Toda la información guardada por la etiqueta head es considerada meta-información, porque es información que habla de la página pero visualmente no es vista por el usuario, es información utilizada por los algoritmos dedicados a posicionar las páginas en los buscadores.

Todo lo que queramos que se vea en el cuerpo de la página debe escribirse en la etiqueta body, como lo dice su nombre, es el cuerpo de la página.

Es importante respetar la jerarquía de las etiquetas, no podemos por ejemplo abrir la etiqueta body dentro del head y cerrarla fuera del body. Son etiquetas que están al mismo nivel y van separadas, esto ocurre con todas las etiquetas. HTML al no ser un lenguaje de programación no nos mostrará errores, pero cuando tenemos algo mal cerrado podría no llegar a mostrarse correctamente.

Otro aspecto importante en HTML es la indentación, es el espaciado que creamos con la tecla tab y nos muestra claramente la jerarquía de las etiquetas, cómo están posicionadas unas dentro de otras.

En el siguiente ejemplo veremos una etiqueta sin indentación seguida de otra con indentación. 

            
                <h1>
                <p>
                Este texto no tiene indentación, no se ve claramente cómo es la jerarquía.
                </p>
                </h1>
            
        
            
                <h1>
                    <p>
                        Este texto si tiene indentación, se ve claramente cómo es la jerarquía.
                    </p>
                </h1>

Comentarios en HTML

Un comentario en HTML o en lenguajes de programación como Javascript son textos o notas que no queremos que el navegador interprete, nos sirven para dejar indicaciones para los programadores. En HTML los comentarios inician con menor que (<), signo de exclamación y dos guiones (-), y los cerramos con dos guiones(-) y un mayor que (>).

Es posible convertir en comentario cualquier parte de un código por medio de atajos, en el caso de VScode debemos seleccionar el código o texto a convertir en comentario y presionar Ctrl+K seguido de Ctrl+C.

Etiqueta HTML y el atributo lang

La etiqueta HTML es la que se encarga de contener la cabeza y cuerpo de nuestro documento (head y body). Desde HTML 5 es importante por cuestiones de posicionamiento de contenido, agregar el idioma en el cuál nosotros vamos a escribir el contenido de nuestro documento HTML.

El atributo lang tiene por defecto el valor en que significa inglés por english, aunque podemos ser específicos con el lenguaje hasta el punto de especificar la región pues hay ciertas expresiones que son propias de una región, no es lo mismo español de españa que español de mexico o de Venezuela, por ejemplo. Para japonés se usa jp, para español utilizamos es, esto lo podemos buscar en internet.

Si llegasemos a necesitar señalar que el español es de algún área en específico debemos usar el guion (-) seguido del dominio del país en mayúsculas, por ejemplo VE en el caso de Venezuela. Esto es solo en casos muy especiales donde el contexto del contenido es muy particular de algún país en específico.

Etiqueta DOCTYPE

Antes de HTML 5 existieron las versiones anteriores de HTML y una mezcla entre HTML y XML que es otro lenguaje de marcado, en cada una de estas versiones había diferentes tipos de documento, por lo cuál la etiqueta DOCTYPE resultaba ser muy larga y engorrosa, porque la etiqueta iba acompañada de una URL hacia la especificación del tipo de documento. Esto ya no se utiliza, pero antes dependiendo del tipo de documento que especificabamos para escribir, algunas etiquetas eran soportadas y otras no o el comportamiento y visualización de las mismas se modificaba.

Con HTML 5 la W3C se estableció que solo iba a haber un solo tipo de documento, y para especificar que es un tipo de documento con la nueva versión simplente hay que colocar <!DOCTYPE html>

Metaetiquetas básicas

<meta charset="UTF-8">

Charset es el juego de caracteres en el cual está codificada una página (esto tiene que ver con los idiomas), UTF-8 es el juego de caracteres universal estándar, por ejemplo hay caracteres que no se utililzan en inglés y solo en español como la letra ñ. Sin la etiqueta <meta charset="UTF-8"> la letra Ñ y los acentos no se representarían de forma correcta, incluso algunas dieresis. Esta etiqueta permite que estos caracteres se muestren de forma correcta en el navegador. Por esta razón es muy importante agregar esta metaetiqueta <meta charset="UTF-8">. Algunos navegadores ya reconocen estos caracteres pero su adición forma parte de las buenas prácticas.

Los caracteres especiales se pueden codificar igualmente sin necesidad de la etiqueta <meta charset="UTF-8">, de hecho lo estamos haciendo para poder agregar los picoparéntesis sin alterar el código, pero esto si es que se quiere hacer en algún navegador viejo, como tal no es una buena práctica y lleva mucho más trabajo. En el siguiente link podrás ver cómo escribir los caracteres especiales en HTML: Caracteres especiales en HTML.

<meta http-equiv="X-UA-Compatible" content:"IE=edge">

Esta etiqueta no es estándar e incluso de puede borrar, pero se utiliza para indicar al navegador Internet Explorer que trate de igualar la visualización del documento a la brindada por edge. Como ya no se utiliza IE se puede quitar, pero VScode lo agrega porque este es un programa desarrollado por Microsoft.

<meta name="viewport" content="width=device-width initial-scale=1.0">

Esta metaetiqueta tiene dos atributos, name que tiene como valor el nombre de la etiqueta y content que es el contenido al que hace referencia. El viewport es la parte visible del navegador donde se dibuja o renderiza el contenido HTML. Esta etiqueta es muy importante porque le dice a los dispositivos cómo es que se va a adaptar el contenido del documento, para el responsive-web-design.

Esta metaetiqueta tiene varios atributos posibles, es recomendable ver toda la información en MDN, sin embargo, los más importantes son width=device-width, con eso indica que el ancho del documento HTML se adaptará al ancho del dispositivo.

Seguidamente aparece el atributo initial-scale, el cual es utilizado para indicar la escala inicial a la cual se va a comenzar el contenido, el valor 1.0 indica 100%. Esto es muy importante esta etiqueta para el responsive-web-design, aunque escribamos todo el código CSS necesario para que la página se adapte de forma adecuada a distintos dispositivos, no funcionará todo ese código si la etiqueta <meta name="viewport" content="width=device-width initial-scale=1.0"> no está presente.

<title>>

El título o title es el nombre del documento HTML, debe ser un título referente al contenido que va a encontrar el usuario en la página web que está visitando. Es importante que el título esté entre los 55 y 65 caracteres de longitud (podemos saber cuántos caracteres tiene un texto que escribamos al subrayarlo, esta información aparece en la barra inferior de VScode).

<meta name = "description" content = "Descripción de la página">

La descriptión o metaetiqueta description contendrá la descripción de la página web, la descripción se escribe como valor del atributo content. El texto de la descripción no debe sobrepasar los 165 caracteres de longitud.

La longitud del texto colocado en las etiquetas <title> y <meta name = "description" content = "Descripción de la página"> es importante porque ambas etiquetas nos ayudarán en el SEO, estos textos de muestran en Google al buscar una página, el título donde hacemos clic normalmente para ingresar es <title> y la descripción que le sigue es la etiqueta <meta name = "description" content = "Descripción de la página">. La cantidad de caracteres mencionada es la que cabe justamente en la visualización del buscador, si sobrepasamos esta cantidad de caracteres apareceran puntos suspensivos en la descripción o el título mostrado en los enlaces del buscador.

También es importante dejad a todas las páginas del sitio web el mismo título y la misma descripción, es importante definir esto de forma adecuada para cada documento HTML, si no, todas las páginas podrían decir que van al Home del sitio web, por ejemplo, cuando en realidad no es así. Todo esto es importante para el SEO de nuestros proyectos. Esto afecta mucho cuando Google muestra un sublistado de las secciones que según Google son más visitadas dentro de un sitio web. Si tenemos los mismos títulos y descripción se verán enlaces y descripciones iguales en cada enlace.

La etiqueta <title> se reflejará visualmente en la pestaña del navegador, es el texto que aparece en ella.

Encabezados

Estas etiquetas van desde <h1> a <h6>, los encabezados nos permiten titularizar nuestro contenido. Los niveles de encabezado generan una jerarquía semántica de contenido, pero además muestran diferencias visuales respecto a su tamaño como se puede ver en los siguientes ejemplos. Esto de la jerarquía generada por los niveles de los encabezados es por SEO, en cada documento HTML solo debe haber un solo título h1, porque cuando los algoritmos de los buscadores van y revisan nuestro código HTML de nuestro sitio o aplicaciones, siempre se fijan en el h1 para detectar cuál es el título importante de nuestro HTML. Aunque HTML no es un lenguaje de programación y no lanzará un error por tener mas de un título h1, una buena práctica es tener un solo título de este tipo para que los algoritmos sepan cuál es el texto principal.

Con el resto de etiquetas de título no sucede lo mismo, podemos tener tantos títulos <h2>, <h3>, <h4> ,<h5> y <h6> dependiendo de las necesidades que tengamos a la hora de crear nuestro contenido.

Mientras menor el número de la etiqueta de encabezado, mayor importancia tendrá el contenido, siendo h1 el más relevante y el h6 el menos relevante. SI se llega a necesitar una etiqueta más allá del h6, no se podrá hacer nada y habrá que distribuir de mejor manera los encabezados.

Es muy importante no tener texto suelto en una etiqueta, de preferencia siempre debe estar dentro de una etiqueta.

Podemos generar párrafor de prueba al escribir lorem, esto es una herramienta brindada por emmet. Etiqueta b

Esta etiqueta nos permite convertir el texto en negrita, la b es de bold.

Etiqueta i

Esta etiqueta nos permite convertir el texto en itálica o cursiva negrita, i es de italic.

Etiqueta sup

Esta etiqueta nos permite represetar textos como un superíndice, E=MC2, por ejemplo.

Etiqueta sub

Esta etiqueta nos permite represetar textos como un subíndice, H2O, por ejemplo.

Etiqueta u

Esta etiqueta nos permite subrayar texto, sin embargo, esto se recomienda hacerlo con CSS.

Etiqueta mark

Esta etiqueta nos permite subrayar texto pero con color amarillo, ocurre lo mismo que con la u, normalmente esto se recomienda hacerlo con CSS.

Etiqueta small

Esta etiqueta nos permite representar el texto como las letras pequeñas de los términos y condiciones, por ejemplo.

Etiquetas de texto semánticas

Estas etiquetas aparte de darnos una referencia visual, también nos van a dar un significado.

Etiqueta strong

Esta etiqueta se utiliza para dar importancia a un texto, pero visualmente se ve como si hubiesemos utilizado la etiqueta bold.

Etiqueta em

Esta etiqueta nos permite hacer énfasis a un texto, pero visualmente se ve como si hubiesemos utilizado la etiqueta i.

Etiquetas blockquote y cite

Esta etiqueta la utilizamos para hacer citas en bloque, es decir, en párrafo (citas grandes), si deseamos hacer citas de algún artículo de una página por ejemplo, esta etiqueta puede llevar el atributo cite para indicar la fuente de la cita. Pero también podemos utilizar la etiqueta cite seguida el blockquote, que funciona para definir la fuente de una obra creativa. La etiqueta blockquote agrega una indentación en el texto.
